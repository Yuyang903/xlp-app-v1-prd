享乐派App_v1.0 产品需求文档 (PRD)

产品名称：享乐派

文档版本：v1.0

编写日期：2026-03-30

1. 产品概述

享乐派App_v1.0 是一款专注于音乐实体专辑数字化体验与粉丝互动的移动端应用。
用户可以通过 NFC 或扫描二维码将购买的实体专辑绑定至 App，获取专属的数字专辑内容（包括音乐、MV、电子写真、数字小卡、专属徽章等），
并通过权益机制参与互动活动（如赠品领取资格、线下活动抽奖等）。

2. 核心概念

核心资产与权益概念：

  • 1. 收藏：用户通过绑定NFC卡或二维码，获取数字资产。
    • 1-1. 内容类：包含音乐、MV、写真等数字内容。
      • 1-1-1. 音乐：专辑内的无损音频资源。
      • 1-1-2. MV：专属的MV、VLOG、花絮等视频。
      • 1-1-3. 写真：专属艺人高清写真图片。
    • 1-2. 增值类：具有收藏与展示价值的数字周边。
      • 1-2-1. 小卡：随实体专辑附赠的纯数字收藏卡，或实物NFC卡感应识别后获得的数字收藏卡。
      • 1-2-2. 徽章：用户完成特定条件（如线下打卡、线上参与活动等）后获得的数字荣誉标识。
  • 2. 权益：用户持有特定数字资产后所解锁的附加福利，包括持续发布的各种互动类的线上线下活动（如赠品领取资格、线下活动抽奖等）。

3. 核心资产关联关系说明

3.1 核心名词解释

  • NFC卡 / 二维码：物理介质，通常内置于实体专辑或作为独立周边发售，是用户连接并激活数字资产的“钥匙”。
    注：每一张 NFC 芯片或二维码都具有唯一序列号，一旦被某账号扫码/碰触激活绑定后，该实体介质即与该账号强绑定，不可被其他账号再次激活。二维码的业务原理与 NFC 卡完全一致，仅仅是识别方式（相机扫码 vs 射频感应）的不同。
  • 商品：购买单元（SKU）。例如“陶喆《STUPID POP SONGS》平装版”或“黄子弘凡《除了快乐禁止入内》A平台特典版”。
  • 收藏品：数字资产的顶层容器。对应某个具体的音乐专辑或项目，包含该专辑下的所有音乐、MV、写真、社区权限以及专属的小卡图鉴和徽章墙。
  • 数字小卡：隶属于特定“收藏品”下的数字卡片资产，收集小卡可点亮对应的图鉴，可链接关联线上活动。
  • 徽章：荣誉与打卡凭证。用户完成特定任务（如线下NFC打卡、参与线上活动等）后获得的数字标识。特定的徽章或徽章组（如集齐7个子徽章合成完整徽章）可作为解锁高级权益的凭证。
  • 权益：附加福利与特权。用户持有特定的数字资产（如收藏品、数字小卡、徽章等）后所解锁的专属福利资格，包含但不限于实物赠品领取、线下活动抽奖、优先购买权及专属内容访问权限等。

3.2 NFC 的链接种类与激活路径

根据使用场景和绑定逻辑的不同，NFC 芯片在系统中分为以下两种链接种类：

  1. 唯一绑定 NFC
    • 说明：芯片具有强身份属性，App感应后可与用户账号进行唯一绑定。
    • 适配场景：需要进行身份唯一性绑定的实体卡，例如专辑配置的专属小卡、独立购买的实体小卡等。
    • 激活路径与资产获取（2种模式）：
      • (1) 绑定收藏品，同时获取该收藏品配置的“默认小卡”
        • 路径：NFC卡 → 商品 → 收藏品
        • 说明：随新版实体专辑发行的标准卡。卡片与特定“商品”绑定。用户激活后，获得该商品对应的“数字收藏品（完整版）”，解锁该收藏品下的音乐、MV、写真及社区权限等。（注：多个商品可能映射到同一个收藏品，如不同平台的特典版商品，激活后获得的是相同的数字收藏品）。
        • 小卡获取逻辑：在该模式下，若专辑系统配置有“默认小卡”（例如全员普赠的 N 张基础卡），用户在获得收藏品的同时，将自动同步获取这些默认小卡（一次可获取多张）。此时小卡的发放是由“收藏品”规则触发，而非与 NFC 卡直接建立强关联。
      • (2) 绑定收藏品，同时仅获取该 NFC 对应的“单一特定小卡”
        • 路径：NFC卡 → 商品 → 收藏品 → 小卡（唯一绑定）
        • 说明：作为独立周边或专辑配置形式发行的特殊卡（如带有唯一绑定属性的实体小卡盲盒、特典卡或历史旧版卡）。
        • 小卡获取逻辑：卡片激活后，用户首先获得小卡对应商品映射的完整“数字收藏品”。同时，该 NFC 卡与某一张特定的“数字小卡”具有唯一绑定关系，用户在获得收藏品的同时，仅会获取与该物理介质一一对应的那一张特定数字小卡，将其点亮在收藏品的小卡图鉴中。
  2. 公用 NFC
    • 说明：芯片不与单一用户绑定，支持被多个不同的用户重复感应识别。（一般用于徽章收集）
    • 适配场景：线下打卡活动。将 NFC 芯片固定在特定的线下打卡点（如演唱会场馆、快闪店等），用户打开 App靠近 NFC 即可完成打卡。
    • 打卡产物：用户打卡后获取的数字资产具有灵活性，可以是特定的“数字小卡”、“专属徽章”、“数字收藏品”，也可以直接跳转至一个特定的“活动专题链接”。
    • 扩展玩法（徽章组合）：在线下或线上的打卡活动中，可设置多个“子徽章”收集任务。当用户通过公用 NFC 陆续打卡集齐指定的 N 个（例如 7 个）子徽章后，系统会自动将这些子徽章组合为一个“完整高级徽章”（合成后原有的子徽章在界面上取消显示或转化为组成部分）。用户获得该完整徽章后，可通过点击徽章直接跳转至特定专题页，参与抽奖或领取实体/虚拟奖品。

4. 核心功能架构

App 主要分为以下几个核心模块：

  • 登录/注册模块：支持验证码、密码登录，以及首次登录的资料完善。
  • 首页模块：展示用户的数字专辑柜/收藏，支持添加新专辑。
  • 识别添加模块：通过 NFC 感应或扫描二维码绑定实体专辑。
  • 专辑详情模块：数字专辑的核心消费场景，包含音乐播放、MV、写真、数字小卡、徽章展示及专属社区。
  • 消息通知模块：接收系统通知与社区互动消息。
  • 个人中心模块：管理个人资料、账号设置、问题反馈等。

5. 功能模块详细说明

5.1 登录与注册/找回密码

为提供更直观的交互体验，该模块已制作成高保真交互原型，请直接在下方操作体验：

功能说明：

  1. 本机号码一键登录：App 检测到本机号码后，作为首屏默认登录方式，一键快速认证。
  2. 验证码登录：用户输入手机号获取验证码进行快捷登录。
  3. 密码登录：老用户可使用手机号+密码进行登录。
  4. 第三方登录：支持微信、微博快捷授权登录。
  5. 首次登录设置：新用户首次通过验证码登录后，需设置登录密码，并进入资料完善流程（如设置昵称、头像、性别、生日等）。
  6. 找回密码：用户忘记密码时，可通过手机号验证码进行密码重置。
  7. 协议强校验：所有登录方式均要求勾选用户协议，未勾选时拦截并弹窗提示确认。

5.2 首页 (数字专辑柜)

为提供更直观的交互体验，该模块已制作成高保真交互原型，请直接在下方操作体验：

功能说明：

  1. 专辑展示：以书架或列表形式展示用户已绑定/收藏的数字专辑。
  2. 空状态引导：当用户无收藏专辑时，展示缺省页并引导用户添加。
  3. 添加专辑入口：提供明显的“添加”按钮，点击后弹出添加方式选项（NFC 或 扫码）。
  4. 添加成功反馈：成功绑定新专辑后，首页需给出明确的视觉反馈与动效。

5.3 识别与添加模块

功能说明：

  1. NFC 识别：调用手机 NFC 功能，提示用户将手机靠近实体专辑内置的 NFC 芯片进行感应绑定。
  2. 扫码识别：调用相机权限，扫描实体专辑上的专属二维码进行绑定。
  3. 异常处理：若识别码无效、已被绑定或读取失败，弹出明确的错误提示。

5.4 专辑详情与消费模块 (核心)

为提供更直观的交互体验，该模块已制作成高保真交互原型，请直接在下方操作体验：

功能说明：

进入具体的数字专辑后，展示专辑维度的所有专属内容，按 Tab 分类：

  • 5.4.1 专辑主页 (音乐)：展示曲目列表，支持点击播放。下方包含长篇的专辑介绍。
  • 5.4.2 MV：展示专辑关联的官方音乐视频列表。
  • 5.4.3 写真：展示高清电子写真图集或海报。
  • 5.4.4 徽章：展示该专辑关联的线上/线下活动徽章墙，已点亮的徽章高亮显示。
  • 5.4.5 小卡：展示随专辑附赠或后续抽取的数字小卡，未获得的小卡呈锁定状态。
  • 5.4.6 社区：该专辑的专属粉丝交流区，支持按最热/最新排序，支持发布图文动态。
  • 5.4.7 权益：官方公告区。用于发送跟该收藏品强相关的事务通知或者专属活动通知（如线下见面会、专属抽奖等）；内容展示形式支持图文混排或纯文字。

5.5 消息通知模块

功能说明：

  1. 消息列表：集中展示系统通知（如上新、活动）、互动消息（评论、点赞等）。
  2. 空状态：当没有消息时展示占位缺省图。

5.6 个人中心模块

功能说明：

  1. 未登录状态：展示默认头像与登录引导，限制部分功能的使用，点击触发登录提示。
  2. 个人资料管理：支持修改头像、昵称、性别、生日（调用日期选择器）等信息。
  3. 设置与账号管理：提供解绑设备/账号相关的操作及二次确认提示。
  4. 问题反馈：提供表单让用户提交使用过程中遇到的问题（支持文字描述、图片上传等），帮助优化产品。

6. 后续规划与建议

  • 数据埋点：建议在 NFC 识别成功率、核心页面停留时长、播放器使用率上做好数据埋点。
  • 性能与并发预估：
    • 大文件加载（音频/MV/写真）：由于数字资产包含大量无损音频、高清视频（MV）及高分辨率电子写真，预计单用户单次消费产生的下行带宽峰值较高。需引入阿里云 CDN 加速节点分发，依托其海量全网边缘节点与超大带宽储备，理论上可承载百万级用户同时在线播放无损音频或观看高清 MV，有效应对明星发售新专等极端峰值场景。同时，在 App 端需做好合理的分级缓存策略（如音乐文件的边播边存、图片缩略图预加载等），以保障播放流畅度并避免占用用户过多本地存储空间。
    • NFC 感应识别并发：在演唱会、线下快闪店等大型“公用 NFC”打卡场景中，预计会在极短时间内（如开场前后 1 小时内）产生瞬时高并发的识别打卡与资产下发请求。系统需具备弹性扩容能力，接口层应支持高并发处理，并通过消息队列等机制削峰填谷，确保现场用户“秒级响应”的打卡体验。
