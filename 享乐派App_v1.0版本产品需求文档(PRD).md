# 享乐派App_v1.0 产品需求文档 (PRD)

## 1. 产品概述
享乐派App_v1.0 是一款专注于音乐实体专辑数字化体验与粉丝社区互动的移动端应用。用户可以通过 NFC 或扫描二维码将购买的实体专辑绑定至 App，获取专属的数字专辑内容（包括音乐、MV、电子写真、数字小卡、专属徽章等），并在专属的专辑社区中与其他粉丝进行互动。

## 2. 核心功能架构
App 主要分为以下几个核心模块：
- **登录/注册模块**：支持验证码、密码登录，以及首次登录的资料完善。
- **首页模块**：展示用户的数字专辑柜/收藏，支持添加新专辑。
- **识别添加模块**：通过 NFC 感应或扫描二维码绑定实体专辑。
- **专辑详情模块**：数字专辑的核心消费场景，包含音乐播放、MV、写真、数字小卡、徽章展示及专属社区。
- **消息通知模块**：接收系统通知与社区互动消息。
- **个人中心模块**：管理个人资料、账号设置、问题反馈等。

## 3. 功能模块详细说明

### 3.1 登录与注册/找回密码
**关联设计稿：**
- `11登录-验证码登录-1` ~ `14-登录-验证码登录-4`
- `15-登录-首次登录设置密码`
- `16-登录-密码登录-1`
- `17-找回密码-1`, `18-找回密码-2`
- `19-首次登录资料完善-1`, `20-首次登录资料完善-2`

**功能说明：**
1. **验证码登录**：用户输入手机号获取验证码进行快捷登录。
2. **密码登录**：老用户可使用手机号+密码进行登录。
3. **首次登录设置**：新用户首次通过验证码登录后，需设置登录密码，并进入资料完善流程（如设置昵称、头像、性别、生日等）。
4. **找回密码**：用户忘记密码时，可通过手机号验证码进行密码重置。

### 3.2 首页 (数字专辑柜)
**关联设计稿：**
- `0-首页`
- `1-首页-无收藏状态`
- `2-首页-添加选项`
- `5-首页-添加选项-添加成功`

**功能说明：**
1. **专辑展示**：以书架或列表形式展示用户已绑定/收藏的数字专辑。
2. **空状态引导**：当用户无收藏专辑时，展示缺省页并引导用户添加。
3. **添加专辑入口**：提供明显的“添加”按钮，点击后弹出添加方式选项（NFC 或 扫码）。
4. **添加成功反馈**：成功绑定新专辑后，首页需给出明确的视觉反馈与动效。

### 3.3 识别与添加模块
**关联设计稿：**
- `3-NFC识别`
- `4-扫描二维码`
- `22-识别码错误提示`

**功能说明：**
1. **NFC 识别**：调用手机 NFC 功能，提示用户将手机靠近实体专辑内置的 NFC 芯片进行感应绑定。
2. **扫码识别**：调用相机权限，扫描实体专辑上的专属二维码进行绑定。
3. **异常处理**：若识别码无效、已被绑定或读取失败，弹出明确的错误提示（如 `22-识别码错误提示`）。

### 3.4 专辑详情与消费模块 (核心)
**关联设计稿：**
- **专辑主页**：`7-专辑详情-专辑`, `8-专辑详情-下滑样式`
- **徽章系统**：`9-专辑详情-徽章`, `10专辑详情-徽章-点击放大`, `11专辑详情-徽章-获得新徽章`
- **数字小卡**：`12-专辑详情-小卡`, `13-专辑详情-卡片-点击放大`, `14-专辑详情-卡片-获得新卡片`
- **视觉内容**：`15-专辑详情-写真`, `16-专辑详情-MV`
- **音乐播放**：`19-专辑详情-播放界面`, `20-专辑详情-播放界面-查看歌词`, `21-专辑详情-播放界面-播放列表`
- **专属社区**：`17-专辑详情-社区-1`, `18-专辑详情-社区-2`

**功能说明：**
1. **专辑主页**：展示专辑封面、简介、曲目列表等基本信息，支持下拉交互查看更多内容。
2. **徽章系统**：展示该专辑专属的数字徽章。用户绑定或达成特定条件可获得新徽章，支持点击放大查看徽章细节与获取时间。
3. **数字小卡**：展示随专辑附赠的数字小卡（盲盒机制或固定获取），支持点击放大欣赏，获得新卡片时有专属动效。
4. **数字写真与MV**：提供高清电子写真画册浏览和专属 MV 视频播放。
5. **音乐播放器**：
   - 包含播放/暂停、上一首/下一首、进度条控制。
   - 支持呼出播放列表，进行曲目切换。
   - 支持查看同步滚动的歌词。
6. **专属社区**：仅限拥有该专辑（或特定权限）的用户进入的讨论区，用户可发帖、评论、点赞，增强粉丝互动。

### 3.5 消息通知模块
**关联设计稿：**
- `6-消息通知`
- `6-1-消息通知-无消息状态`

**功能说明：**
1. **消息列表**：集中展示系统通知（如上新、活动）、互动消息（评论、点赞等）。
2. **空状态**：当没有消息时展示占位缺省图。

### 3.6 个人中心模块
**关联设计稿：**
- `1-个人中心-首页`
- `2-个人中心-首页-未登录`
- `3-个人中心-个人资料`, `4-个人中心-个人资料-2`, `5-个人中心-个人资料-日期选择器`
- `7-个人中心-解绑提示`
- `8-个人中心-登录提示`
- `9-个人中心-问题反馈`, `10-个人中心-问题反馈-2`

**功能说明：**
1. **未登录状态**：展示默认头像与登录引导，限制部分功能的使用，点击触发登录提示。
2. **个人资料管理**：支持修改头像、昵称、性别、生日（调用日期选择器）等信息。
3. **设置与账号管理**：提供解绑设备/账号相关的操作及二次确认提示。
4. **问题反馈**：提供表单让用户提交使用过程中遇到的问题（支持文字描述、图片上传等），帮助优化产品。

---

## 4. 后续规划与建议
- **数据埋点**：建议在 NFC 识别成功率、核心页面停留时长、播放器使用率上做好数据埋点。
- **性能优化**：由于包含大量高清写真、小卡及音频视频内容，需做好缓存管理和加载优化，避免占用过多手机存储。
